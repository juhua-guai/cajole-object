# 📝 Web 版 PRD：布布的嘴替日常

| 属性 | 内容 |
| :--- | :--- |
| 项目 | 布布的嘴替日常（Web 演示版） |
| 技术栈 | 前端 React/Next，后端 Python（复用现有业务逻辑） |
| 目标 | 本地演示沉浸式恋爱互动，复刻作死/求生模式与建议卡体验 |
| 配置 | 前端本地存储 API Key/Base URL/Model，可在线修改即时生效 |
| 会话 | 单用户、内存上下文（刷新/重启即清空），不做持久化 |
| 部署 | 本地运行，演示为主 |

---

## 1. 产品概述
- 定义：浏览器端“布布 & 一二”沉浸式对话应用，主打懒人式点选互动与情绪驱动模式切换。
- 核心价值：开屏即破冰、A/B/C 情绪梯度建议卡、作死/求生双模式反差萌、熊言熊语的强 IP 代入。
- 不做：多用户隔离、云端存储、复杂权限；仅演示核心交互。

## 2. 用户与场景
- 目标用户：喜欢 IP 互动、需要轻量情绪陪伴的普通用户。
- 使用场景：本地打开网页即玩；桌面和移动端均需正常体验。

## 3. 功能清单
- 配置管理
  - 设置弹层：编辑 API Key/Base URL/Model；提示本地存储安全风险；支持保存、重置。
  - 配置生效：保存后立即应用于后端调用（无需刷新）。
- 冷启动
  - 进入页面即按时间段生成破冰消息（贪吃/自恋话题）并返回 A/B/C 建议卡。
- 建议卡交互
  - 3 张卡固定情绪梯度：A 开心/撒娇，B 中性，C 暴怒。
  - 点击后立即进入回复流程并显示 loading 态。
- 手动输入
  - 文本框折叠/展开；发送后先做情绪识别（关键词优先，短文本默认 B，长文本可 LLM 判定）→ 映射为 A/B/C。
- 模式逻辑
  - 当前模式：作死 roast / 求生 survival。
  - 自动切换：在作死模式下连续 3 次 C 进入求生；在求生模式下连续 3 次 A 退出；非对应情绪重置计数。
  - 手动请求：模式开关仅“申请”切换；若情绪不足则 pending，后续情绪满足后自动切换。
- 对话上下文
  - 保存最近 N 轮（默认 10）用于 LLM 上下文；提供“清空会话”入口。
- 资源兜底
  - 头像圆形裁剪；作死/求生两套布布头像；加载失败使用占位图；情绪图标用于建议卡。
- 异常提示
  - LLM 限流/余额不足提示；情绪识别失败回退 B；输入乱码使用预设熊言熊语兜底文案。

## 4. 交互流程（核心路径）
1) 进入页面 → 读取本地配置 → 调用冷启动 → 展示布布回复 + A/B/C 卡。
2) 用户点击建议卡：
   - 记录选择（A/B/C）→ 更新计数与模式 → 生成布布新回复 + 建议卡 → 如模式变化，更新 UI。
3) 用户手动输入：
   - 情绪识别 → 映射 A/B/C → 与建议卡同样流程。
4) 用户点击模式开关：
   - 提交 target_mode → 若情绪基础不足返回 pending 提示；满足时即时切换并提示。
5) 设置弹层：
   - 编辑配置 → 保存（写 LocalStorage）→ 前端后续请求使用新配置；可重置恢复默认空值。

## 5. 模式与计数规则
- 作死模式：贱萌、阴阳怪气。
- 求生模式：认错、卑微哄。
- 切换触发：
  - roast → survival：连续 3 次情绪 C；若期间出现 A/B，愤怒计数清零。
  - survival → roast：连续 3 次情绪 A；若期间出现 B/C，开心计数清零。
- 手动请求：
  - 期望 survival 但愤怒计数不足 → pending，等待后续 C；满足后切换并清空期望。
  - 期望 roast 但开心计数不足 → 同理 pending，等待后续 A。

## 6. 信息架构与前端分层
- Header：模式指示/切换按钮、设置入口。
- ChatArea：气泡流（左布布、右一二），头像圆形裁剪。
- SuggestionBar：A/B/C 卡片，含情绪图标与文案，点击态与 loading 态。
- InputBar：折叠/展开的手动输入框。
- Modal/Toast：配置弹层、错误提示、模式提示。
- 状态（前端）：`mode`、`desiredMode`、`happyCount`、`angryCount`、`history`（滑窗）、`config`（LocalStorage）、`loading/error`。

## 7. 后端接口（本地演示版，Python）
- `POST /api/chat/cold-start` → `{ reply, suggestions }`
- `POST /api/chat/reply` `{ text, choice? }` → `{ reply, suggestions, mode_changed?, current_mode }`
- `POST /api/chat/mode-toggle` `{ target_mode }` → `{ accepted, current_mode, reason }`（reason 枚举：`already_in_mode` / `pending_emotion_confirm` / `emotion_supported`）
- 内部复用：`StateManager`（模式/计数）、`LLMService`（上下文窗口+JSON 解析）、`EmotionAnalyzer`（关键词优先+LLM 兜底）。
- 会话：单用户内存，不持久化；刷新/重启清空。

## 8. 配置与存储策略
- LocalStorage 键：`bubu-config`（示例字段：`apiKey`、`model`）。
- 默认值：空字符串 / 兼容默认模型名；缺失时提示用户补填。
- 安全提示：在设置弹层提醒“仅本地存储，不上传服务器”。
- 生效策略：保存后新请求立即使用新配置；无需刷新。

## 9. 资源规范
- 头像：`/static/avatars/bubu_roast.png`、`bubu_survival.png`、`yier.png`；失败时占位图。
- 情绪图标：`/static/icons/icon_happy.png`、`icon_neutral.png`、`icon_furious.png`。
- 占位图：灰色默认头像/损坏占位；保持与现有资源命名一致。

## 10. 异常与兜底
- LLM 调用失败/限流：提示“额度/速率受限，请检查 key/model”。
- 情绪识别失败：回退情绪 B。
- 输入乱码：固定熊言熊语吐槽文案。
- 资源加载失败：使用占位图，界面不崩。

## 11. 里程碑（演示版）
- M1：前端骨架 + 设置弹层 + 冷启动展示 + 建议卡点击。
- M2：完整模式逻辑（自动/手动）+ 情绪识别 + LLM 接入。
- M3：异常兜底、资源占位、移动端适配与基础样式优化。
